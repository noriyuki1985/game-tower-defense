# CHANGELOG

## v4.0.2 — 主人公攻撃復旧・改善提案反映

- v4.0.1 のコア再設計後、主人公の近接攻撃処理が存在しなくなっていた問題を修正。
- `BAL.kingAttackRange` / `kingAttackDamage` / `kingAttackCd` / `kingAttackWindup` を追加し、王の自動斬撃をバランス調整しやすい定数へ集約。
- 王の攻撃時に向き補正、斬撃表示、バースト、リング、浮き文字を追加。
- `damageEnemy()` に source 引数を追加し、王の攻撃は旧仕様同様に装甲軽減を受けにくい処理にした。

## v4.0.1 — 起動バグ修正

- ページ読込時、ゲーム開始前から描画ループが回り始め、状態 `S` が `null` のまま `update()` が `S.status` を参照して初回フレームで例外発生 → ループが停止し、「出陣する」を押しても何も起きない不具合を修正。
- 対応：`update()` を null セーフ化（`if(!S || S.status!=="playing") return;`）し、ループ側も `S` が存在する時だけ更新するよう変更。起動→アイドル→開始→実行まで例外ゼロを確認。

## v4.0.0 — コア再設計（core rebuild）

v3.12.4 で感じていた「ゲームとしての物足りなさ」を、表面の調整ではなくコアの仕組みから作り直した版。
単一ファイル（`index.html`）構成へ移行し、既存アセットを流用。

### 設計の作り直し

- **建物配置を「役割タグ付き離散スロット」に変更**
  道スロット＝柵のみ／高所スロット＝弓塔・大砲・兵舎／金鉱スロット＝金鉱のみ。
  → 「道の真ん中に弓塔が建つ」問題を構造的に解消。塔は必ず道を見下ろす位置に座る。

- **お金の仕組みを刷新（連続吸い上げを廃止）**
  スロットでアクションを選ぶと費用を **その場で一括消費**。残高不足なら実行不可。
  建設／強化／修理を1回行うとそのスロットは **ロック**し、王が離れて戻るまで再操作できない。
  → 「レベルアップしたら一旦お金を使うのをやめる」を仕組みとして実装。立ちっぱなしで財布が溶ける挙動を撤廃。

- **経済と戦闘の択を作った**
  金鉱はウェーブ突破後に一括収入。早い投資ほど後半が伸びる。ただし前線から離れた3か所にしか建たず、建設／強化の間は主戦場を空ける（機会費用）。
  弓塔・大砲は王が近いと発射が速くなる。前線に立つか、後方で経済を回すか――王の位置取りが戦略になる。
  強化費は段階ごとに逓増（×1.6）するため全施設をMAXにはできず、毎ウェーブ投資先を選ぶ。

- **マップを一新**
  2ルート（主街道・側道）が中央で合流して城へ向かう構成。合流点が自然な殺し間になる。
  背景は手続き描画（草グラデ＋深度ブロブ＋柔らかい道＋木・岩＋ドリフトする霧）で、王や建物のトゥーン絵柄と画風を統一。単調さを解消。

### 演出

- レベルアップ＝ヒットストップ＋段階色の衝撃波リング＋スプライトのスウィープ切替＋粒子＋画面フラッシュ＆シェイク＋上昇音。Lv4は専用バーストと持続グロー。
- 王＝移動方向へのリーン／スクワッシュ＆ストレッチ／足元の砂煙、待機の呼吸、建設ポーズ。
- 敵＝歩行ボブ／被弾フラッシュ＆ノックバック／撃破バースト／コインドロップ。
- 兵士＝攻撃ランジ（攻撃フレーム流用）。

### コンテンツ

- 建物5種（柵・弓塔・大砲・兵舎・金鉱）×各4レベル（既存Lv1-4画像を流用）。
- 敵10種を流用、8ウェーブの難易度アーク。最終ウェーブは将軍ボス。
- localStorage に最高到達ウェーブと音設定のみ保存。**ゲーム内通信ゼロ**（詳細は README）。

### 確認済みの既知メモ

- 数値（コスト・収入・HP・ウェーブ物量など）は初期値。実プレイ後の調整を前提に `BAL` / `STAT` / `DEF` / `WAVES` に集約してある。
- 高所スロットは道から 46〜74px 脇に配置（塔の射程 132〜158 で道を確実にカバー）。道スロットはルート上（0.6〜3.3px）に配置し柵が確実に食い止める。
- `file://` 直開きは画像が出ない場合があるため、ローカル確認はサーバ経由を推奨（README参照）。
