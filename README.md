## 遊び方

初めての場合は `王立訓練場` を選ぶと、説明多めで柵・弓塔・金鉱・王の采配ボーナスを確認できる。通常マップではゲーム中説明を抑え、実戦だけに集中する。

- 盤面をタップ／クリックで **王が移動**。少し離れた敵には王が自動で弓攻撃する。
- **道スロット（柵マーク）** には柵だけ建てられる。敵を食い止め、火力施設の攻撃時間を作る。
- **高所スロット（塔マーク）** には弓塔・大砲・兵舎を建てて道を狙い撃つ。
- **金鉱スロット（鉱マーク）** は前線から離れた場所にあり、ウェーブ突破後に金を生む。
- スロットに立つとメニューが出る。**費用はその場で一括**。建設／強化／修理を1回行うと、**その床を離れて戻るまで再操作できない**（使いすぎ防止）。
- 弓塔・大砲は **王が近くにいるとき発射が速くなる**（王の采配ボーナス）。前線に立つか、金鉱を伸ばしに後方へ走るか――その判断がゲームの核。

### 操作

| 操作 | 内容 |
| --- | --- |
| タップ／クリック | 王の移動先指定・メニュー選択 |
| 上部 ☰ メニュー | 速度、一時停止、音、ステージ再選択、やり直し |
| スペース／P | 一時停止 |
| F | 速度切替（1x / 2x） |
| M | サウンド ON/OFF |

## ステージ選択

`v4.2.0` で、v4コアのステージを6種類へ拡張した。開始画面で以下を選択できる。

| ステージ | 区分 | 特徴 |
| --- | --- | --- |
| meadow / 草地の前線 | 本番固定 | 2ルートの基準ステージ。柵と弓塔の基本を確認する。 |
| crossroads / 分岐路の前線 | 本番固定 | 3ルートが合流する。分岐対応と誤配置を検証する。 |
| ridge / 高台の前線 | 本番固定 | 高台の3ルート。合流前に削り、中央坂道の柵を維持する。 |
| rivergate / 橋門の前線 | 本番固定 | 橋に敵が集中する。柵と大砲で橋上を制圧する。 |
| forest / 森陰の前線 | 本番固定 | 曲がり道が多く走り兵が抜けやすい。王の移動と兵舎配置を試す。 |
| bastion / 廃砦の前線 | 本番固定 | ルート数は少ないが重兵と破城槌が多い高難度ステージ。 |

ステージごとに、ルート、建設スロット、ウェーブ構成、初期資金、敵倍率、敵圧が切り替わる。開始画面では初期資金、敵HP倍率、敵圧、建設床数、ルート別敵数を短く表示する。v4.8.0では全ステージを「本番固定」扱いにし、新要素追加前の基準値として初期資金・敵倍率・敵数・出現間隔・後半HP上昇を固定した。

## 敵歩行アニメーション

- `grunt / brute` は重めの歩行アニメーション
- `runner / saboteur` は速い走行アニメーション
- `shield` は盾を構えた前進アニメーション
- `bomber` は爆弾を持った走行アニメーション
- `shaman` は杖と衣装が揺れる歩行アニメーション

## 実行とデプロイ

ブラウザのセキュリティ上、画像は同一オリジンから読み込む必要があるため、`file://` で直接開くと一部ブラウザで画像が表示されないことがある。ローカル確認時は簡易サーバ経由が確実。

```powershell
# 例: このフォルダ直下で
python -m http.server 8080
# → http://localhost:8080/ を開く
```

GitHub Pages では、リポジトリ直下にこの `index.html` と `assets/` を置けばそのまま公開される。

```
  assets/

```

## 通信仕様（ゲーム内通信ゼロ）

本ゲームは **ゲーム内通信を一切行わない**。`index.html` を静的検査した結果は次のとおり（v4.0.2 時点）。

| 項目 | 使用 |
| --- | --- |
| `fetch` | なし |
| `XMLHttpRequest` | なし |
| `WebSocket` | なし |
| `navigator.sendBeacon` | なし |
| `EventSource` | なし |
| `iframe` 埋め込み | なし |
| 外部 `script` 読込 | なし |
| 外部 URL（http/https） | なし |
| Google Workspace / Drive / Gmail / Calendar API | なし |
| OAuth 関連 | なし |
| 任意ファイル読取・アップロード | なし |

データ保存は `localStorage`（最高到達ウェーブと音設定のみ）。音は WebAudio による合成音で、音源ファイルの取得も行わない。

GitHub Pages 等で公開した場合に発生するのは、静的サイト配信として必要な **`index.html` / 画像（PNG）の取得** のみで、これはゲーム内通信ではなくアセット読込である。
