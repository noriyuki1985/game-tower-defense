# King Builder Defense — v4.6.2

王みずから前線を駆け、道を柵で塞ぎ、脇の高所に塔を並べて守るタワーディフェンス。
HTML / CSS / JavaScript のみ。単一の `index.html` で動作し、GitHub Pages 等の静的配信に対応する。


## v4.6.2 / v4.6.1 の主な更新

- 主人公の `idle / run / bow attack / hit` 全フレームで白フチを再補正し、透過の見え方を統一。
- 主人公描画を「同一キャンバスの足元位置固定」で扱うよう調整し、攻撃・被弾時のサイズぶれを軽減。
- 弓攻撃アニメーションを専用4フレームで再生するよう戻し、攻撃中も立ち姿と同系統の見え方に統一。
- 弓塔 / 大砲 / 兵舎 / 柵 / 採掘所の描画倍率を引き上げ、戦場での視認性を改善。

## 遊び方

- 盤面をタップ／クリックで **王が移動**。少し離れた敵には王が自動で弓攻撃する。
- **道スロット（柵マーク）** には柵だけ建てられる。敵を食い止め、火力施設の攻撃時間を作る。
- **高所スロット（塔マーク）** には弓塔・大砲・兵舎を建てて道を狙い撃つ。
- **金鉱スロット（鉱マーク）** は前線から離れた場所にあり、ウェーブ突破後に金を生む。
- スロットに立つとメニューが出る。**費用はその場で一括**。建設／強化／修理を1回行うと、**その床を離れて戻るまで再操作できない**（使いすぎ防止）。
- 弓塔・大砲は **王が近くにいるとき発射が速くなる**（王の采配ボーナス）。前線に立つか、金鉱を伸ばしに後方へ走るか――その判断がゲームの核。

### 操作

| 操作 | 内容 |
| --- | --- |
| タップ／クリック | 王の移動先指定・メニュー選択 |
| 上部 ☰ メニュー | 速度、一時停止、音、ステージ再選択、やり直し |
| スペース／P | 一時停止 |
| F | 速度切替（1x / 2x） |
| M | サウンド ON/OFF |

## ステージ選択

`v4.2.0` で、v4コアのステージを6種類へ拡張した。開始画面で以下を選択できる。

| ステージ | 区分 | 特徴 |
| --- | --- | --- |
| meadow / 草地の前線 | 本番候補 | 2ルートの基準ステージ。柵と弓塔の基本を確認する。 |
| crossroads / 分岐路の前線 | 検証用 | 3ルートが合流する。分岐対応と誤配置を検証する。 |
| ridge / 高台の前線 | 本番候補 | 高台の3ルート。合流前に削り、中央坂道の柵を維持する。 |
| rivergate / 橋門の前線 | 本番候補 | 橋に敵が集中する。柵と大砲で橋上を制圧する。 |
| forest / 森陰の前線 | 本番候補 | 曲がり道が多く走り兵が抜けやすい。王の移動と兵舎配置を試す。 |
| bastion / 廃砦の前線 | 本番候補 | ルート数は少ないが重兵と破城槌が多い高難度ステージ。 |

ステージごとに、ルート、建設スロット、ウェーブ構成、初期資金、敵倍率が切り替わる。開始画面では初期資金、敵HP倍率、建設床数、ルート別敵数を短く表示する。

## 設置メニューの誤操作対策

`v4.0.5` で建設メニューを調整した。

- 建設・強化・修理ボタンを大きくした。
- メニュー周辺のタップ/クリックが移動指示に化けにくいよう、安全領域を追加した。
- 建設操作直後の短時間入力を無視し、二重入力や誤爆を抑制した。
- 金不足時のメッセージは短く「金不足」と表示する。


## 敵歩行アニメーション

`v4.3.1` で、敵キャラクターを静止画表示から4フレーム歩行シート表示へ切り替えた。

- `grunt / brute` は重めの歩行アニメーション
- `runner / saboteur` は速い走行アニメーション
- `shield` は盾を構えた前進アニメーション
- `bomber` は爆弾を持った走行アニメーション
- `shaman` は杖と衣装が揺れる歩行アニメーション
- `siege` は車輪と車体が動く攻城車アニメーション
- `warlord / overlord` は重装ボス系の歩行アニメーション

各敵の静止画はフォールバックとして残し、歩行シートの読み込みに失敗した場合でもゲームは継続する。

## 実行とデプロイ

ブラウザのセキュリティ上、画像は同一オリジンから読み込む必要があるため、`file://` で直接開くと一部ブラウザで画像が表示されないことがある。ローカル確認時は簡易サーバ経由が確実。

```powershell
# 例: このフォルダ直下で
python -m http.server 8080
# → http://localhost:8080/ を開く
```

GitHub Pages では、リポジトリ直下にこの `index.html` と `assets/` を置けばそのまま公開される。

## 構成

```
game-tower-defense/

  README.md
  CHANGELOG.md

    IMAGE-PROMPTS-for-ChatGPT.md      # 画像生成用の画風指針と指示文
  assets/
    images/...        # 既存アセットを流用（王・建物Lv1-4・敵・エフェクト・UI）
```

## 通信仕様（ゲーム内通信ゼロ）

本ゲームは **ゲーム内通信を一切行わない**。`index.html` を静的検査した結果は次のとおり（v4.0.2 時点）。

| 項目 | 使用 |
| --- | --- |
| `fetch` | なし |
| `XMLHttpRequest` | なし |
| `WebSocket` | なし |
| `navigator.sendBeacon` | なし |
| `EventSource` | なし |
| `iframe` 埋め込み | なし |
| 外部 `script` 読込 | なし |
| 外部 URL（http/https） | なし |
| Google Workspace / Drive / Gmail / Calendar API | なし |
| OAuth 関連 | なし |
| 任意ファイル読取・アップロード | なし |

データ保存は `localStorage`（最高到達ウェーブと音設定のみ）。音は WebAudio による合成音で、音源ファイルの取得も行わない。

GitHub Pages 等で公開した場合に発生するのは、静的サイト配信として必要な **`index.html` / 画像（PNG）の取得** のみで、これはゲーム内通信ではなくアセット読込である。
