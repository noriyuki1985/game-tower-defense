# King Builder Defense — 設計仕様 v4.0.0

この文書は、v3.12.4 の不満点を起点にコアを作り直した v4.0.0 の設計をまとめたもの。
実装（`index.html`）と対応している。数値は初期値で、調整は `BAL` / `STAT` / `DEF` / `E` / `WAVES` に集約。

## 1. 何を直したか（不満点 → 対応）

| 元の不満 | 原因 | v4.0.0 の対応 |
| --- | --- | --- |
| 背景と王の絵柄が合わない | 背景の絵柄が王のトゥーンと別物 | 背景を手続き描画で作り直し、王・建物のトゥーンと統一 |
| 道の真ん中に弓塔が建つ | 配置床が型を事前固定 | 役割タグ付きスロット（道＝柵のみ／高所＝塔） |
| バランスが単調 | 受動経済＋連続消費で択がない | 一括消費・逓増コスト・ウェーブ収入で毎回選択が生まれる |
| キャラの動きが乏しい | 静的スプライト中心 | 王・敵・兵士に手続きアニメ（リーン／ボブ／ランジ等） |
| レベルアップ演出が弱い | 控えめなFXのみ | ヒットストップ＋衝撃波＋スウィープ＋粒子＋音 |
| お金を一気に使ってしまう | 床に立つと連続吸い上げ | 一括消費＋「1訪問1アクション」ロック |
| 経済が戦略にならない | 金鉱が受動トリクル | 金鉱＝離れた場所のウェーブ収入（機会費用と択） |

## 2. コンセプト

王は盤上に1体だけ。**王が物理的にどこにいるか**が、防衛・経済・采配のすべてに効く。
道を柵で塞ぎ、脇の高所から塔で撃ち、後方の金鉱で経済を回す。毎ウェーブ「いま武装か、経済か」を選ぶ。

## 3. ステージ（1枚を作り込み）

- キャンバス 480×800 縦。城は左上。敵は右下（主街道）と左下（側道）から出て、中央 `MERGE(240,452)` で合流し城へ。
- 合流点が自然な殺し間。城前の最終直線がもう一つの防衛線。
- 背景は手続き描画：草グラデ、深度ブロブ、柔らかい2本の道（合流プラザ）、木・岩のprop（影＋微揺れ）、ドリフトする霧。

## 4. スロットと建物（役割タグ）

スロットに立つと、その種別で許可された建物だけが選べる。

| スロット種別 | 許可建物 | 配置の考え方 |
| --- | --- | --- |
| 道（road） | 柵 | ルート上（0.6〜3.3px）。敵を食い止める |
| 高所（ground） | 弓塔・大砲・兵舎 | 道の脇 46〜74px。道を見下ろす |
| 金鉱（economy） | 金鉱 | 前線から遠い3か所（ルートから167〜229px） |

建物（初期値・各4Lv、既存Lv1-4画像を流用）:

- **柵**：道に置く壁。範囲内の敵を減速し、接触で削られる（修理可）。火力施設の攻撃時間を作る。
- **弓塔**：長射程の単体火力（射程158 / CD560ms / Dmg18〜）。走り兵・抜けた敵向け。
- **大砲**：低速・範囲（射程132 / CD1480ms / Dmg52 / 範囲64〜）。合流点の群れ向け。
- **兵舎**：兵士を生成し前線を物理的に埋める（Lvで人数・HP・火力増）。側道・複数ルート向け。
- **金鉱**：ウェーブ突破後に収入（22〜）。脆く、前線から遠い。

レベル倍率（`STAT`）：Dmg ×[1,1.65,2.4,3.3] / 射程 ×[1,1.1,1.2,1.3] / CD ×[1,0.9,0.82,0.74] / HP ×[1,1.55,2.2,3.0] / 収入 ×[1,1.75,2.7,3.9]。

## 5. お金（経済と戦闘の択）＝本丸

- **一括消費**：アクション確定で費用を一括で引く。残高不足は実行不可（連続吸い上げ廃止）。
- **1訪問1アクション**：建設／強化／修理を1回するとスロットがロック。王が範囲外（半径30px）に出て戻るまで再操作不可。
  → レベルアップ後に勝手に使い続けない。1回ごとに「これでいいか」を考える間が生まれる。
- **逓増コスト**：強化費は Lvごとに ×1.6。全部はMAXにできない。
- **収入源**：敵ドロップ（王が歩いて回収・約9秒で消える）＋ 金鉱（ウェーブ突破時に一括付与）。
- **位置取りの緊張**：弓塔・大砲は王が半径96px内にいると発射 +30%。前線に立てば火力が上がるが、その間は金鉱を伸ばせない。金鉱は遠いので、建てる間は前線が薄くなる。

## 6. 敵（10種を流用）

雑兵・走り兵・重兵・盾兵（装甲）・破壊兵（施設を狙う）・爆弾兵（撃破時に施設へ爆発）・呪術兵（周囲を回復）・破城槌（高HP・装甲）・将軍／大将軍（ボス・周囲強化オーラ）。
装甲は被ダメージを割合軽減。ボスはHPバーと撃破演出を強化。

## 7. ウェーブ（8段の難易度アーク）

主街道のみ → 側道追加 → 走り兵の速攻 → 合流乱戦（大砲が活きる）→ 破壊兵・爆弾兵（施設狙い）→ 盾兵・重兵の本隊 → 三方向＋呪術兵 → 最終決戦（破城槌＋将軍ボス）。
ウェーブ間は小休止（初回2.6秒 / 以降4.2秒）。各ウェーブの狙いは画面下のヒントで提示。

## 8. 演出

- **レベルアップ**：ヒットストップ110ms → 段階色（青→緑→黄→赤）の衝撃波リング → スプライトのスウィープ切替 → 粒子＋上昇火花 → 画面フラッシュ＆シェイク → 上昇音。Lv4は専用バースト＋持続グロー＋バナー。
- **王**：移動でrunフレーム＋スクワッシュ＆ストレッチ＋リーン＋砂煙、待機で呼吸＋idle切替、建設で専用ポーズとリング。移動先マーカー表示。
- **敵**：歩行ボブ、被弾フラッシュ＆ノックバック、撃破バースト、コインドロップ。
- **兵士**：攻撃ランジ（攻撃フレーム流用）。

## 9. 技術・制約

- 単一 `index.html`（HTML+CSS+JS内包）。Canvas2D。外部ライブラリなし。
- 既存 `assets/` を相対参照で流用。
- 保存は `localStorage`（最高到達ウェーブ・音設定のみ）。**ゲーム内通信ゼロ**。
- 内部解像度は固定480×800、CSSでアスペクト比を保って画面にフィット（モバイル対応）。

## 10. 調整の入口（チューニングするならここ）

- 難易度：`WAVES`（物量・間隔・ルート・遅延）と `spawnEnemy` のlate ramp。
- 経済：`BAL.startCoins` / 各 `DEF.cost` `baseUpgrade` / `DEF.mine.income` / `STAT.income`。
- 位置取り感：`BAL.kingRally` `kingRallyBonus`、スロット座標 `SLOTS`、ルート `ROUTE`。
- 「使いすぎ防止」の強さ：`BAL.slotRadius`（離れて戻る距離）。

## 12. v4.0.2 追加メモ

- 王は半径 `BAL.kingAttackRange` 内の敵を自動で斬る。
- 攻撃値・射程・クールダウンは `BAL.kingAttackDamage` / `BAL.kingAttackRange` / `BAL.kingAttackCd` に集約した。
- 王の攻撃は主火力ではなく、前線に立つ意味を補強する副次火力として扱う。
- 今後、王が強すぎる場合はダメージよりも射程・クールダウンを先に調整する。

## v4.0.5 / v4.1.0 追記

### 設置メニュー誤操作対策

- 建設メニューはボタンを大きくし、メニュー全体に安全領域を持たせる。
- メニュー内またはスロット近傍のタップ/クリックは移動指示に変換しない。
- 建設・強化・修理直後は短時間だけ追加入力を無視し、二重入力や誤爆を抑制する。
- 金不足メッセージは短く「金不足」とし、表示時間も短くする。

### v4コアへのステージ選択復帰

- `meadow` / `crossroads` / `ridge` を `STAGE_DEFS` に集約し、ルート、スロット、ウェーブ、初期資金、敵倍率をステージごとに保持する。
- 実行時は選択ステージを `setActiveStage()` で `ROUTE` / `SLOTS` / `WAVES` に反映する。
- v4の単一HTML構成は維持し、v3系のステージ選択・診断表示の考え方だけを戻す。
- 診断表示は、初期資金、敵HP倍率、建設床数、ルート別敵数に限定し、説明過多にならないよう短く表示する。


## v4.2.0 追記 — ステージ拡張

`v4.2.0` では、v4コアに追加ステージ3種を実装した。画像生成は行わず、既存素材・Canvas手続き描画・ステージ別ルート定義で差分を作る。

### 追加ステージ

- `rivergate` / 橋門の前線
  - 3ルートが橋周辺へ集中する。
  - 柵と大砲の役割を確認しやすい。
  - 橋・川はCanvas上の簡易地形演出で表現する。

- `forest` / 森陰の前線
  - 曲がり道が多く、走り兵・破壊兵が抜けやすい。
  - 王の移動、弓塔配置、兵舎の前線維持を試す。
  - 森の影は背景色と半透明ブロブで表現する。

- `bastion` / 廃砦の前線
  - 2ルートだが重兵・盾兵・破城槌が多い。
  - 火力不足と柵修理判断を診断しやすい。
  - 廃砦の壁・石畳はCanvas上の簡易地形演出で表現する。

### 設計方針

- 新ステージ追加時は `ROUTES` / `SLOTS` / `WAVES` / `STAGE_DEFS` の4点を追加する。
- ステージ選択UIは `STAGE_DEFS` を列挙するため、追加したステージは自動で選択肢に出る。
- 画像を増やす前に、ルート構造とゲーム性の差分を先に検証する。


## v4.3.1: Stage map progression

The game now starts from a stage-select map rather than a long instruction card. The purpose is to make the game feel like a multi-stage campaign and to let the player advance continuously after clearing each stage.

### Rules

- `STAGE_ORDER` defines the campaign order.
- `meadow` is unlocked by default.
- Clearing a stage records it in `clearedStages`.
- The next stage is added to `unlockedStages`.
- The result screen offers direct progression to the next stage.
- Progress is persisted in `localStorage` under `kbd-v4-save`.
- The stage-select map is CSS/DOM only. No new image assets are required.


## v4.3.1 敵アニメーション設計

敵は従来の静止画 `assets/images/enemies/enemy_*.png` をフォールバックとして残しつつ、通常描画では `assets/images/enemies/walk/enemy_*_walk.png` の4フレームシートを使う。

- 1シート = 横4フレーム
- 描画時に `spriteFrame()` で該当フレームだけを切り出す
- `enemy.anim` を各敵ごとに保持し、移動更新中に進める
- `runner / saboteur` は速め、`siege` は遅め、ボス系は重めのアニメーション速度にする
- 読み込み失敗時は従来の静止画描画へフォールバックする

これにより、敵が滑っているように見える問題を軽減し、進軍中の存在感を上げる。


## v4.3.3 敵攻撃・被弾・破壊演出

- 敵は `assets/images/enemies/attack/enemy_<type>_attack.png` を持ち、施設や柵に接敵している間は攻撃シートを4分割再生する。
- 被弾時は `stun` を使って短い硬直を与え、通常描画の上から `lighter` / `screen` 合成で白フラッシュを重ねて視認性を上げる。
- `siege` 撃破時は `assets/images/enemies/destroy/enemy_siege_destroy.png` を4分割再生し、木片色のパーティクルと画面シェイクを重ねる。
- 歩行シートは継続利用し、攻撃していない時間帯だけ歩行アニメーションを表示する。


## v4.3.4 主人公の弓攻撃

- 主人公は短距離弓兵として振る舞う。
- 射程は `BAL.kingAttackRange` を使用し、従来の近接斬撃より長い。
- 攻撃開始時に `attackWindup` をセットし、予備動作後に `S.projectiles` へ王専用の矢を追加する。
- 描画は `assets/images/king/attack/king_attack_bow.png` の4フレームを `spriteFrame` で再生する。
- 既存の移動 / 待機アニメは維持しつつ、攻撃中のみ弓アニメへ切り替える。


## v4.3.5 主人公の被弾リアクション

- 主人公にHPは持たせず、敵との近接接触時に短い被弾リアクションだけを発生させる。
- `assets/images/king/hit/king_hit.png` の4フレームを使い、`drawKing()` で被弾中に優先表示する。
- `updateKingHurt()` が接触判定を行い、短いノックバック、フラッシュ、リング演出を発生させる。
- 被弾中は `updateKingAttack()` を止め、見た目の分かりやすさを優先する。


## v4.3.6 主人公の待機 / 移動アニメ統一

- `assets/images/king/idle/king_idle_1.png` 〜 `king_idle_4.png` を新規生成画像へ差し替える。

- `drawKing()` の待機処理は2フレーム切替から4フレーム循環へ変更する。
- 主人公は弓兵として見えるよう、待機・移動の両方で弓を保持した姿に統一する。
