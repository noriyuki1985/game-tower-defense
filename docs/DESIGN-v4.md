# King Builder Defense — 設計仕様 v4.0.0

この文書は、v3.12.4 の不満点を起点にコアを作り直した v4.0.0 の設計をまとめたもの。
実装（`index.html`）と対応している。数値は初期値で、調整は `BAL` / `STAT` / `DEF` / `E` / `WAVES` に集約。

## 1. 何を直したか（不満点 → 対応）

| 元の不満 | 原因 | v4.0.0 の対応 |
| --- | --- | --- |
| 背景と王の絵柄が合わない | 背景の絵柄が王のトゥーンと別物 | 背景を手続き描画で作り直し、王・建物のトゥーンと統一 |
| 道の真ん中に弓塔が建つ | 配置床が型を事前固定 | 役割タグ付きスロット（道＝柵のみ／高所＝塔） |
| バランスが単調 | 受動経済＋連続消費で択がない | 一括消費・逓増コスト・ウェーブ収入で毎回選択が生まれる |
| キャラの動きが乏しい | 静的スプライト中心 | 王・敵・兵士に手続きアニメ（リーン／ボブ／ランジ等） |
| レベルアップ演出が弱い | 控えめなFXのみ | ヒットストップ＋衝撃波＋スウィープ＋粒子＋音 |
| お金を一気に使ってしまう | 床に立つと連続吸い上げ | 一括消費＋「1訪問1アクション」ロック |
| 経済が戦略にならない | 金鉱が受動トリクル | 金鉱＝離れた場所のウェーブ収入（機会費用と択） |

## 2. コンセプト

王は盤上に1体だけ。**王が物理的にどこにいるか**が、防衛・経済・采配のすべてに効く。
道を柵で塞ぎ、脇の高所から塔で撃ち、後方の金鉱で経済を回す。毎ウェーブ「いま武装か、経済か」を選ぶ。

## 3. ステージ（1枚を作り込み）

- キャンバス 480×800 縦。城は左上。敵は右下（主街道）と左下（側道）から出て、中央 `MERGE(240,452)` で合流し城へ。
- 合流点が自然な殺し間。城前の最終直線がもう一つの防衛線。
- 背景は手続き描画：草グラデ、深度ブロブ、柔らかい2本の道（合流プラザ）、木・岩のprop（影＋微揺れ）、ドリフトする霧。

## 4. スロットと建物（役割タグ）

スロットに立つと、その種別で許可された建物だけが選べる。

| スロット種別 | 許可建物 | 配置の考え方 |
| --- | --- | --- |
| 道（road） | 柵 | ルート上（0.6〜3.3px）。敵を食い止める |
| 高所（ground） | 弓塔・大砲・兵舎 | 道の脇 46〜74px。道を見下ろす |
| 金鉱（economy） | 金鉱 | 前線から遠い3か所（ルートから167〜229px） |

建物（初期値・各4Lv、既存Lv1-4画像を流用）:

- **柵**：道に置く壁。範囲内の敵を減速し、接触で削られる（修理可）。火力施設の攻撃時間を作る。
- **弓塔**：長射程の単体火力（射程158 / CD560ms / Dmg18〜）。走り兵・抜けた敵向け。
- **大砲**：低速・範囲（射程132 / CD1480ms / Dmg52 / 範囲64〜）。合流点の群れ向け。
- **兵舎**：兵士を生成し前線を物理的に埋める（Lvで人数・HP・火力増）。側道・複数ルート向け。
- **金鉱**：ウェーブ突破後に収入（22〜）。脆く、前線から遠い。

レベル倍率（`STAT`）：Dmg ×[1,1.65,2.4,3.3] / 射程 ×[1,1.1,1.2,1.3] / CD ×[1,0.9,0.82,0.74] / HP ×[1,1.55,2.2,3.0] / 収入 ×[1,1.75,2.7,3.9]。

## 5. お金（経済と戦闘の択）＝本丸

- **一括消費**：アクション確定で費用を一括で引く。残高不足は実行不可（連続吸い上げ廃止）。
- **1訪問1アクション**：建設／強化／修理を1回するとスロットがロック。王が範囲外（半径30px）に出て戻るまで再操作不可。
  → レベルアップ後に勝手に使い続けない。1回ごとに「これでいいか」を考える間が生まれる。
- **逓増コスト**：強化費は Lvごとに ×1.6。全部はMAXにできない。
- **収入源**：敵ドロップ（王が歩いて回収・約9秒で消える）＋ 金鉱（ウェーブ突破時に一括付与）。
- **位置取りの緊張**：弓塔・大砲は王が半径96px内にいると発射 +30%。前線に立てば火力が上がるが、その間は金鉱を伸ばせない。金鉱は遠いので、建てる間は前線が薄くなる。

## 6. 敵（10種を流用）

雑兵・走り兵・重兵・盾兵（装甲）・破壊兵（施設を狙う）・爆弾兵（撃破時に施設へ爆発）・呪術兵（周囲を回復）・破城槌（高HP・装甲）・将軍／大将軍（ボス・周囲強化オーラ）。
装甲は被ダメージを割合軽減。ボスはHPバーと撃破演出を強化。

## 7. ウェーブ（8段の難易度アーク）

主街道のみ → 側道追加 → 走り兵の速攻 → 合流乱戦（大砲が活きる）→ 破壊兵・爆弾兵（施設狙い）→ 盾兵・重兵の本隊 → 三方向＋呪術兵 → 最終決戦（破城槌＋将軍ボス）。
ウェーブ間は小休止（初回2.6秒 / 以降4.2秒）。各ウェーブの狙いは画面下のヒントで提示。

## 8. 演出

- **レベルアップ**：ヒットストップ110ms → 段階色（青→緑→黄→赤）の衝撃波リング → スプライトのスウィープ切替 → 粒子＋上昇火花 → 画面フラッシュ＆シェイク → 上昇音。Lv4は専用バースト＋持続グロー＋バナー。
- **王**：移動でrunフレーム＋スクワッシュ＆ストレッチ＋リーン＋砂煙、待機で呼吸＋idle切替、建設で専用ポーズとリング。移動先マーカー表示。
- **敵**：歩行ボブ、被弾フラッシュ＆ノックバック、撃破バースト、コインドロップ。
- **兵士**：攻撃ランジ（攻撃フレーム流用）。

## 9. 技術・制約

- 単一 `index.html`（HTML+CSS+JS内包）。Canvas2D。外部ライブラリなし。
- 既存 `assets/` を相対参照で流用。
- 保存は `localStorage`（最高到達ウェーブ・音設定のみ）。**ゲーム内通信ゼロ**。
- 内部解像度は固定480×800、CSSでアスペクト比を保って画面にフィット（モバイル対応）。

## 10. 調整の入口（チューニングするならここ）

- 難易度：`WAVES`（物量・間隔・ルート・遅延）と `spawnEnemy` のlate ramp。
- 経済：`BAL.startCoins` / 各 `DEF.cost` `baseUpgrade` / `DEF.mine.income` / `STAT.income`。
- 位置取り感：`BAL.kingRally` `kingRallyBonus`、スロット座標 `SLOTS`、ルート `ROUTE`。
- 「使いすぎ防止」の強さ：`BAL.slotRadius`（離れて戻る距離）。

## 12. v4.0.2 追加メモ

- 王は半径 `BAL.kingAttackRange` 内の敵を自動で斬る。
- 攻撃値・射程・クールダウンは `BAL.kingAttackDamage` / `BAL.kingAttackRange` / `BAL.kingAttackCd` に集約した。
- 王の攻撃は主火力ではなく、前線に立つ意味を補強する副次火力として扱う。
- 今後、王が強すぎる場合はダメージよりも射程・クールダウンを先に調整する。
