# King Builder Defense — 画像生成ガイド

本ドキュメントは、v4.0.0 に合わせた画像生成の指針と優先順位付きのChatGPT指示文をまとめたもの。
背景は手続き描画（コード内）で統一されたため、王・建物のリスタイルと不足の演出スプライトが対象。

## 画風バイブル

**見下ろし角 3/4 の絵本トゥーン**。上左からの暖色日中光。以下に沿って生成すること。

### カラーパレット（彩度絞り）
- 王：素焼きのような肌 / 薄紫のマント / 金冠
- 敵：落ち着いた茶〜赤系（各種族で別色、明度を抑える）
- 建物：素朴な木 / 石 / 金属質な鉱山（彩度控えめ）
- 境界：濃い輪郭線（2〜3px、黒〜濃い茶）で塊を引き締め

### 形状と光
- **丸みある塊形状**：角を避け、柔らかな曲線で球面・円筒を意識
- **上左からの日差し**：照らし面は薄い色、陰は濃い色で立体感
- **影＋周囲の環境光反射**：足元の影は紫〜青寄りで浮かせ

### 実装
すべて **背景透明の PNG** で出力。ゲーム内で重ねるため。

## 新規作成する画像（優先順位順）

### 【最高優先】王の新ポーズ・アニメーション

**ユースケース**：王が建物を建てている最中 / 待機時の呼吸と仕草

**既存**：
- idle_1, idle_2（待機静止フレーム2種）
- run_1-4（走行4フレーム）
- king_down（倒れ）
- king_hurt（被弾）

**新規作成（以下の指示をChatGPTに1回のプロンプトで まとめて指示）**：

```
## 王のキャラクター拡張フレーム（3/4見下ろしトゥーン）

【スタイル】
- 上左からの暖色光、濃い輪郭線（2-3px）、丸みのある塊形
- キャラクター：素焼き肌 / 薄紫のマント / 金冠 / 装甲感は控えめ
- 480x800キャンバス内で王を想定スケール（高さ160-180px程度）で描く

【フレーム仕様】
1. **king_building_pose.png** 
   - 王が建物を建てている途中。手を上にやや伸ばし、建築の工夫をしている表情。
   - 背景：透明（PNG）
   - 縦向きに合わせたポーズ。足腰安定、上半身は作業モード。

2. **king_idle_breath_up.png** / **king_idle_breath_down.png**
   - 待機時の呼吸アニメーション用フレーム。
   - up：やや胸を張り、息を吸い込んでいる感じ。
   - down：力を抜き、息を吐いている感じ。
   - idle_1/idle_2 と同じ立ちポーズベースだが、微妙な胸部・肩の上下差を表現。

3. **king_turn_left.png** / **king_turn_right.png**
   - 移動時の向き変わりフレーム。次の移動方向に向き直している最中。
   - ターンアニメーションの補間用（任意）。

【配置・用途】
- すべて assets/images/king/ に配置予定。
- img.src = 'assets/images/king/{ファイル名}.png' で読込。
- 王の現在アニメーション状態に応じて、JS側で `sprite(keyName, kingX, kingY, height, {flip, sx, sy, ...})` で描画。

【カラー参考】
- 肌：#E8D4C0 (素焼き調) ± 影の茶 #9A7C6B
- マント：#9A7FCC (薄紫)
- 装甲：#B8A882 (落ち着いた金)
- 影：周囲は紫〜青寄り (#6B4C8A 等)
- 光：上左から白っぽく
```

### 【高優先】背景オプション（手続き描画の補完）

**現状**：背景は code 内で手続き描画（草グラデ・ブロブ・道・prop）。木・岩 PNG も既存で配置済み。

**追加の場合**：王と画風が統一された **オプション背景セット**（新規デザインの背景に一新したい場合）

```
## 背景セット（任意・手続き描画補完用）

【スタイル】
- 480x800 キャンバス全体
- 3/4見下ろし（上方はより遠景、下方は近景の草）
- 王と同じ光源（上左の暖色日中光）
- 彩度：落ち着いた緑・茶・土色（ビビッドは避ける）

【フレーム例】
1. **bg_meadow_custom_base.png**
   - 草原の地面テクスチャベース。
   - 上方は淡い緑（遠景）、下方は濃い緑＋土（近景）。
   - グラデーション自然なグロス感で、2本のルートが視認しやすいニュートラルベース。

2. **bg_meadow_custom_path.png**
   - 道の描画（既存の手続き描画と統合可能）。
   - または既存 part_road_straight_soft / part_road_curve_soft を使用。

3. **bg_meadow_custom_deco_back.png** / **bg_meadow_custom_deco_front.png**
   - 背景の装飾レイヤー（後ろ・前）。木・岩・石のランドマーク。
   - 奥行き感を強調。

※ 手続き描画で背景統一と判断したため、不足しなければ不要。
```

### 【中優先】演出スプライト（不足分）

既存の爆発・hit・upgrade は3フレーム。不足がなければそのままでOK。

**追加する場合**：

```
## 追加演出スプライト（任意）

【高優先度】

1. **fx_levelup_ring_1/2/3.png**
   - レベルアップ時の段階色衝撃波リング。
   - 色：Lv1=青 / Lv2=緑 / Lv3=黄 / Lv4=赤。
   - 中心からしわが広がるリング（3段階フレーム）。

2. **fx_heal_sparkle_1/2/3.png**
   - 修理完了時の輝き。
   - 淡い青〜白のきらめき粒子。

【低優先度（現状の3フレームで足りたら不要）】

3. **fx_coin_burst_1/2/3.png**
   - コイン回収時の爆発。より金色・光の強調版。

4. **fx_building_complete_1/2/3.png**
   - 建物完成時の専用FX。既存 fx_build を強化版として。
```

### 【低優先】その他（装飾・リスタイル任意）

- **敵キャラの新バリエーション**（既存で足りたら不要）
- **施設の新グラフィック**（既存Lv1-4で足りたら不要）
- **UI 背景パネル**（JS内の矩形で足りたら不要）

## 画像統合の流れ

1. 上記「最高優先」の **王フレーム** を生成・納品
2. `assets/images/king/` に配置（既存と同じ構造）
3. JS側で `IMG_SRC` の `king_building_pose` 等を追加参照し、`sprite()` で使用

---

## 注意点

- **ファイル形式**：すべて **背景透過 PNG**（.png）
- **ファイル名**：半角英数 + アンダースコア（スペースなし）
- **解像度**：実使用スケール（例：王フレームなら高さ160-180pxの想定）を無視せず、見え方優先で。ゲーム側でスケール調整可。
- **色空間**：sRGB 推奨
- **納品形式**：ChatGPT が生成した PNG をそのままダウンロード、ローカルに `assets/images/king/` 等へ上書き配置

---

## 背景について

**v4.0.0 では背景を手続き描画（コード内で描く）したため、王・敵・建物との画風統一が図られている。** 既存背景セットは削除し、代わりにコード内グラデ・prop・パーティクルで表現。

もし背景を「手で書いた絵」に一新したい場合は、上述「背景オプション」セットを依頼し、既存の手続き描画を置き換える選択肢がある。

ただし現状の手続き描画で：
- 背景と王の画風が統一されている ✓
- 単調さが緩和されている（グラデ・prop・霧） ✓

ため、**画像新規作成の優先度は王フレーム >> 背景** と判断している。
